# 微信AI陪伴助手 - 深度优化方案

## 一、需求分析

### 核心目标
1. 集成 less_tokens 库压缩对话历史，节省 40-60% token
2. 扩大上下文窗口（从 20 条扩展到 100+ 条历史）
3. 自动记忆系统，记住用户所有说过的话
4. 优化 prompt 结构，提高 KV Cache 命中率
5. 保持真人对话感，压缩不失自然

### 技术约束
- MiMo API 每次调用有 token 限制
- 对话历史越长，token 消耗越大
- 但上下文越长，AI 回复越像真人

## 二、技术方案

### 2.1 Token 压缩策略
```
消息分级：
- 最近 10 条：完整保留（不压缩）
- 11-30 条：轻度压缩（remove_filler_phrases）
- 31-100 条：中度压缩（remove_filler_phrases + remove_stopwords）
- 100+ 条：摘要替代（只保留摘要）
```

### 2.2 记忆系统架构
```
短期记忆：最近 10 条完整对话
中期记忆：11-100 条压缩对话
长期记忆：用户画像 + 对话摘要
自动记忆：每次对话后自动提取关键信息
```

### 2.3 Prompt 优化
```
固定前缀：角色设定（不变，可缓存）
动态内容：用户摘要 + 压缩历史 + 最新消息
输出约束：简短回复，减少输出 token
```

### 2.4 自动记忆文档
```
存储位置：memory/{user_id}.md
内容：用户姓名、喜好、重要事件、情绪状态
更新频率：每 10 条对话自动更新
```

## 三、实现步骤

1. 安装 less_tokens 库
2. 修改 ai.py 集成压缩
3. 修改 memory.py 扩展存储
4. 创建 auto_memory.py 自动记忆模块
5. 修改 config.py 调整参数
6. 测试验证

## 四、预期效果

- Token 节省：40-60%
- 上下文深度：100+ 条历史
- 记忆持久化：用户画像 + 对话摘要
- 回复质量：保持真人对话感
